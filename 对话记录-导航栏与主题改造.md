## 2. 毛玻璃主题切换滑块

### 目标

取代 `_navbar.md` 中的文字主题链接，改为可视化滑块，点击即切换。

### 修改

**`index.html`** 新增滑块 HTML 结构（独立于导航栏）：

```html
<div class="theme-glass-toggle" id="themeGlassToggle">
  <div class="toggle-track">
    <div class="toggle-thumb" id="toggleThumb">
      <span class="thumb-icon" id="thumbIcon">🌿</span>
    </div>
    <div class="toggle-segments" id="toggleSegments">
      <span class="seg-item" data-theme="vue">🌿</span>
      <span class="seg-item" data-theme="dark">🌙</span>
      <span class="seg-item" data-theme="pure">💧</span>
      <span class="seg-item" data-theme="dolphin">🐬</span>
      <span class="seg-item" data-theme="buble">🫧</span>
    </div>
  </div>
</div>
```

**CSS** 毛玻璃效果核心：

```css
.toggle-track {
  backdrop-filter: blur(12px);
  -webkit-backdrop-filter: blur(12px);
  background: rgba(255, 255, 255, 0.15);
  border: 1px solid rgba(255, 255, 255, 0.25);
  border-radius: 20px;
}
```

**JavaScript** 切换逻辑（`index.html`）：

```javascript
var themes = ['vue', 'dark', 'pure', 'dolphin', 'buble'];
var icons  = ['🌿', '🌙', '💧', '🐬', '🫧'];

function setTheme(index) {
  // 基于实际 segment 坐标计算位置，确保精确对齐
  var firstSeg = segments[0];
  var lastSeg = segments[segments.length - 1];
  var trackRect = track.getBoundingClientRect();
  var firstRect = firstSeg.getBoundingClientRect();
  var lastRect = lastSeg.getBoundingClientRect();
  var firstCenter = (firstRect.left + firstRect.right) / 2 - trackRect.left;
  var lastCenter = (lastRect.left + lastRect.right) / 2 - trackRect.left;
  var step = (lastCenter - firstCenter) / (segments.length - 1);
  thumb.style.left = (firstCenter - 15 + step * index) + 'px';

  thumbIcon.textContent = icons[index];
  // 切换主题 CSS、localStorage 保存
  document.getElementById('theme-style').href = '...' + theme + '.css';
  localStorage.setItem('docsify-theme', theme);
  document.body.classList.remove('theme-vue', 'theme-dark', ...);
  document.body.classList.add('theme-' + theme);
}
```

### 踩坑记录

- **问题**：thumb 位置用 `track.offsetWidth` 计算，但 segments 有 `padding-left: 38px`，导致实际间距不一致
- **修复**：改为用 `getBoundingClientRect()` 获取首尾 segment 的实际坐标来计算步长
- **结论**：涉及 flexbox 布局时，直接用 DOM 坐标计算比数学公式更可靠

## 3. 导航菜单移入自定义导航栏

### 目标

将「首页 / SchoolNetWork / Ownermd」菜单从 docsify 默认的右上角 `.app-nav` 转移到自定义导航栏中，与 Logo 同行。

### 修改

**`index.html`** 中在 Logo 后新增：

```html
<div class="custom-nav-menu">
  <a class="nav-item" href="#/">🏠 首页</a>
  <div class="nav-dropdown">
    <span class="nav-item drop-trigger">SchoolNetWork ▾</span>
    <div class="drop-menu">
      <a href="#/SchoolNetWork/First.md">A</a>
      <a href="#/SchoolNetWork/Second.md">B</a>
    </div>
  </div>
  <!-- ... Ownermd 同理 -->
</div>
```

**隐藏旧的 docsify 导航**：

```css
.app-nav, .app-nav.no-badge { display: none !important; }
```

### 踩坑记录

- **问题**：Docsify 的 `loadNavbar: true` 会检测 `<nav>` 标签并将其隐藏（`display: none`）
- **修复**：用 `<div>` 替代 `<nav>`，class 名避免使用 `nav-menu`（改为 `custom-nav-menu`）
- **结论**：Docsify 对特定标签有内部处理逻辑，避免使用 `<nav>` 作为自定义导航

## 4. 封面页隐藏导航栏

### 目标

封面页显示纯封面内容，不显示顶部导航栏（Logo + 菜单 + 主题滑块）；进入子页面后恢复显示。

### 修改

**`index.html`** JavaScript：

```javascript
function toggleTopBar() {
  var isCover = !location.hash || location.hash === '#/';
  var bar = document.querySelector('.custom-navbar');
  var slider = document.querySelector('.theme-glass-toggle');
  var display = isCover ? 'none' : '';
  if (bar) bar.style.display = display;
  if (slider) slider.style.display = display;
}
window.addEventListener('hashchange', toggleTopBar);
toggleTopBar();
```

## 6. 图片路径修复（绝对路径 → 相对路径）

### 问题

在 md 文件中使用了 Windows 绝对路径：

```markdown
![image.png](E:\Projects\ThersummerBlog\images\xxx.png)
```

浏览器会尝试加载为 `file:///E:/...`，被安全策略拦截。控制台报错：

```
Not allowed to load local resource: file:///E:/Projects/ThersummerBlog/images/...
```

### 修复

改为相对路径（相对于 md 文件位置）：

```markdown
<!-- md 文件在 醒狮社区文档/ 子目录，需 ../ 退回根目录 -->
![描述](../images/个人博客/image.png)
```

### 图片存放规范

```
images/
  ├── blog.png                # 网站图标
  ├── Lionlogo01.png          # Logo
  ├── avatar.jpg              # 封面头像
  ├── 个人博客/                # 博客相关图片
  │   ├── image-xxx.png
  │   └── ...
  └── 醒狮社区/                # 醒狮社区文档图片
      └── xw_xxx.png
```

## 7. 图片尺寸调整 & HTML 标签渲染修复

### 背景

在 Docsify 中，Markdown 原生语法 `![alt](path)` 不支持指定尺寸，必须使用 HTML `<img>` 标签。

### 踩坑记录：HTML 被转义为文本

**现象**：在 md 文件中写下 `<img>` 标签后，页面显示为纯文本而不是图片：

```html
&lt;img src="...png"width ="400" ...&gt;
```

即 `<` 被转义为 `&lt;`，浏览器将其显示为普通文本。

**原因分析**（排查过程）：

1. 用户混合了 Markdown 和 HTML 语法：
   ```markdown
   ![alt]<img src="..."width ="400" ...>
   ```
   — 这是错误的语法混用

2. 即使改为纯 HTML `<img>`，仍有三个写法问题：

| 错误写法 | 问题 | 正确写法 |
|---------|------|---------|
| `png"width` | 属性间无空格 | `png" width` |
| `width ="400"` | `=` 两侧多余空格 | `width="400"` |
| `images\个人博客\` | Windows 反斜杠 | `images/个人博客/` |

3. Docsify 的 Markdown 解析器（marked.js）对不规范的 HTML 属性写法敏感，会将整个标签转为文本

### 最终正确的写法

```html
<img src="../images/个人博客/image-20260702231250853.png" width="400" alt="官方格式" style="border-radius: 8px; box-shadow: 0 2px 8px rgba(0,0,0,0.1);">
```

**规则总结**：

| 规则 | 说明 |
|------|------|
| 使用纯 HTML 标签 | 不要混用 `![]()` Markdown 语法 |
| 属性间留空格 | `<img src="..." width="400" alt="...">` |
| `=` 两侧无空格 | `width="400"` ✔️ / `width ="400"` ❌ |
| 路径用正斜杠 | `images/个人博客/` ✔️ / `images\个人博客\` ❌ |
| `<img` 单独一行 | 不被缩进，前后有空行更安全 |

## 10. 附录：常用操作速查

### 添加新导航项

在 `index.html` 的 `.custom-nav-menu` 中添加：

```html
<a class="nav-item" href="#/新路径">📄 新页面</a>
```

同时同步 `_navbar.md`。

### 隐藏/显示封面导航

修改 `toggleTopBar()` 函数中的条件即可。

### 增删主题

同步修改 3 处：
1. HTML 中的 `.seg-item`
2. JS 中的 `themes` / `icons` 数组
3. JS 中的 `body.classList.remove(...)` 列表

### 在 Docsify 中插入带尺寸图片

```html
<img src="../images/xxx.png" width="400" alt="描述" style="border-radius: 8px;">
```

> ⚠️ 不要混用 Markdown `![]()` 和 HTML `<img>` 语法，两者选其一。

### 路径规范

| 场景 | 正确写法 |
|------|---------|
| md 在根目录 | `images/xxx.png` |
| md 在子目录 | `../images/xxx.png` |
| HTML 标签内 | `../images/xxx.png` |
| 分隔符 | 始终用正斜杠 `/` |
